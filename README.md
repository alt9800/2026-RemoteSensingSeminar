# **リモセン応用セミナー 全6回**

対象：Webエンジニア（GeoJSON・QGIS経験あり、リモートセンシング技術者） 構成：全6回 × 2.5h

標準日程 6/19(金) 6/26(金) 7/3(金) 7/17(金) 7/24(金) 7/31(金)

## **第1回：タイル配信①：データ構造とパース**

現代のWebGISを支えるタイル配信のエコシステムを、FOSSの技術スタックを軸に整理する。 ラスタータイルとベクタータイル（MVT/PMTiles）の違いを実際にタイルサーバーを数種類動かして比較しながら確認し、GISレンダリングエンジンがどのようにタイルを読み込んでいるかをコードを通して理解する。

**到達目標**：現代における地理空間情報の可視化技術とタイル生成のワークフローを説明できるようになること。外部サービスをブラックボックスとして使わず、データの流れを自分の言葉で整理できること。

**キーワード**：XYZ/TMS/WMTS、ラスタータイル、MVT（Mapbox Vector Tile）、PMTiles、protobuf、タイルピラミッド、tippecanoe、MapLibre GL JS、OpenLayers、FOSS4G / OSGeo

**構築するもの**：複数タイルサーバーの比較動作環境、MVT作成スクリプト

---

## **第2回：タイル配信②：自前配信とカートグラフィック**

第1回にて生成したMVT/PMTilesを実際に配信し、スタイルを設計してブラウザ上に描画するまでを完成させる。Nginxによる静的配信の設定、Maputnikでのスタイル設計、MapLibre GL JSによるレンダリングを一貫して実装する。 あわせて、地図スタイリングをサポートするサービス群（MapTiler・Carto・Felt等）とOSMを中心としたオープンデータエコシステムについて整理し、「どのレベルで自作するか」の判断基準を学ぶ。ベアメタルの配信/仮想環境を伴う配信についても考察する。

**到達目標**：外部タイルサービスに依存しない地図配信環境を自分で構築できること。スタイルファイルの構造を理解して編集できること。OSMエコシステムにおける自分たちのデータの位置づけを説明できること。

**キーワード**：MapLibre Style Specification、Maputnik、OpenMapTiles、OpenStreetMap、Martin、MapTiler / Carto / Felt、Nginx、PMTiles静的配信、CORS、ラスタータイルのレンダリングエンジン比較（MapLibre / OpenLayers / Leaflet）

**構築するもの**：Nginx + PMTiles構成の自前タイルサーバー、Maputnikで設計したカスタムスタイルの地図アプ

---


## **第3回：3D技術①：スマホセンサーとAR、位置情報・カメラ・ジャイロの活用**

スマートフォンに搭載されたGPS・ジャイロ・カメラといったセンサーをWebAPIから利用する方法を学ぶ。GeolocationAPIやDeviceOrientationEventの特性と制約を整理した上で、地理座標に紐づいたオブジェクトをカメラ映像に重ねて表示するARアプリを実装する。リモートセンシングデータ由来の観測値をオーバーレイとして可視化する例も示す。

**到達目標**：スマートフォンのセンサーへの理解。地理座標とAR空間の対応関係を理解し、基本的なARアプリを実装できること。

**キーワード**：GeolocationAPI、DeviceOrientationEvent、WebXR Device API、A-Frame、AR.js、カメラパーミッション、センサー精度・デバイス間の差異

**構築するもの**：位置情報連動ARアプリ（観測ポイントのラベル・NDVI値などをカメラ映像に重ねて表示）

---

## **第4回：3D技術②：点群・3DGSのWeb可視化**

WebGISにおける3Dデータの扱いを概観する。主要フォーマット（3D Tiles / glTF / Potree / COPC等）の特性とCesiumJS・Re:Earthなどの新興のGISエンジンの位置づけを整理する。活用事例として文化財・建築・インフラ点検分野での利用を紹介した後、モバイルLiDARスキャンを起点としたパイプライン（CloudCompare → PotreeConverter → deck.gl）を実装する。3DGSについてはSaaSを通じた確認方法とCesiumJSとの統合動向を紹介する。

**到達目標**：WebGISにおける3Dデータのフォーマットと処理パイプラインを説明できること。deck.glで点群をWeb地図上に表示できること。3DGSの現状と活用の入口を把握できること。

**キーワード**：3D Tiles、glTF、Potree / COPC、CesiumJS、Re:Earth（種々のWebGLベースレンダリング）、CloudCompare、PotreeConverter、deck.gl PointCloudLayer、3D Gaussian Splatting、Scaniverse / Luma AI、文化財・インフラ点検事例

**構築するもの**：CloudCompare → Potree形式変換・Web公開、deck.glによる点群の地図オーバーラップ表示

---

## **第5回：高さデータ①：リモートセンシングと高さデータのフォーマット・変換処理**

リモートセンシング技術により高さデータ（DEM/DSM）がどのように生成されているかの背景を理解する。InSARによる干渉処理・LiDAR点群取得・SfMなどの手法とその出力物の特性を概観し、各フォーマット（GeoTIFF / COG / LAS / LAZ）の構造を整理する。 ハンズオンでは配布済みの高さデータ（GeoTIFF）を起点として、GDALを中心とした変換パイプラインを実装する。座標変換・クリッピング・リサンプリングからTerrain RGB形式への変換・タイル書き出しを行う。

**到達目標**：リモートセンシングを起点として高さデータが生成される流れを理解し、WebGISで利用できる形式に変換できること。

**キーワード**：InSAR、LiDAR / 点群、SfM、DSM / DTM / CSM、GeoTIFF、COG（Cloud Optimized GeoTIFF）、LAS / LAZ、GDAL / gdal2tiles / rio-cogeo、Terrain RGB、タイルピラミッド、SRTM / AW3D30 / 基盤地図標高タイル

**構築するもの**：DEM前処理パイプライン、Terrain RGBタイルセット

---

## **第6回：高さデータ②：可視化・実務データとのオーバーラップ・現地調査への応用**

前回処理した高さデータをWebGIS上で3D表示し、リモートセンシング由来のプロダクトや実務データとオーバーラップさせる。MapLibreのterrain機能とdeck.glによる実装を比較しながら実装する。 配布済みのNDWIラスターや防災重点溜池データとのオーバーラップを通じて、実務（防災・土地利用解析）への接続を確認する。 最終的にスマホブラウザで動作する現地調査ツールとして仕上げ、通信前提の設計で十分なユースケースとオフライン設計が必要になるケースの判断基準についても整理する。

**到達目標**：高さデータとリモートセンシング由来データを組み合わせた3D地図をブラウザ上で動かせること。通信前提とオフライン設計のトレードオフを説明できること。

**キーワード**：MapLibre terrain / hillshade、deck.gl TerrainLayer、Terrain RGBデコード、NDWI / NDVI、防災重点溜池（国土数値情報）、垂直誇張、PWA・オフラインキャッシュ（設計論）

**構築するもの**：衛星画像・NDWIオーバーラップ付き3D地形ビューア、防災重点溜池を重ねたハザードマップビューア（ブラウザ向けアプリ）

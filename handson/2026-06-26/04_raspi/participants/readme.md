# デモ①：ラズパイタイルサーバー 受講者向け解説

---

## このデモで体験すること

目の前にある Raspberry Pi 3B+ が、タイル地図の配信サーバーとして動いています。

皆さんのブラウザが取得している地図データは、インターネット上のどこかではなく、**この部屋にある Pi の中**にあります。

---

## なぜラズパイで地図サーバーを立てるのか

Google Maps や国土地理院地図は、常にインターネット経由でタイルを取得しています。電波のない場所では使えません。

地図データをあらかじめ用意して Pi に載せておけば、**電波のない場所でも地図が使えます**。

想定するユースケース：山小屋・農地・災害現場・工事現場など、通信が不安定あるいは圏外の環境での現地調査。Pi を持ち込んでホットスポットを立て、スマートフォンから接続すれば、GPS 位置情報と共に周辺の地図を確認できます。

---

## 今回配信しているデータ

### ラスタータイル（`raster.html`）

OSM のデータから生成した PNG 画像のタイルです。

- 宇部市エリアの道路・森林・河川・水域を描画済み
- ズームレベル z10 / z13 / z16 の三段階、合計 2,058 枚・約 10MB
- `/var/www/html/tiles/{z}/{x}/{y}.png` として静的配信

ブラウザは `GET /tiles/13/7081/3267.png` のように個別にリクエストします。

### ベクタータイル（`vector.html`）

農林水産省の筆ポリゴンデータ（宇部市農地）を PMTiles 形式で配信しています。

- 27,491 フィーチャー（田・畑のポリゴン）、1 ファイル・41MB
- ズームレベル z10–z17
- `land_type` 属性で田（緑）と畑（アンバー）を色分け

ブラウザは PMTiles ファイルの必要な部分だけを **Range Request** で取得します。

---

## ラスタータイルとベクタータイルの違いを観察する

DevTools の Network タブを開いて確認してください。

| | ラスタータイル | ベクタータイル |
|---|---|---|
| リクエスト | 複数の `.png` ファイル | `fude.pmtiles` への Range Request |
| レスポンス | 画像データ | バイナリ（protobuf） |
| レンダリング | 受け取った画像をそのまま表示 | ブラウザが geometry を解釈して描画 |
| スタイル変更 | タイルを再生成しないと変えられない | HTML の style 定義を書き換えるだけ |
| ファイル数 | 2,058 ファイル | 1 ファイル |

ズームしたときの挙動の差も確認してください。ラスタータイルは新しい PNG を取得し、ベクタータイルは取得済みのデータを再レンダリングします。

## 接続方法

### パターン① 会場 WiFi 経由（優先）

当日アナウンスされた IP でアクセスしてください。

### パターン② Tailscale 経由

```
http://100.66.201.10/raster.html
http://100.66.201.10/vector.html
```

### パターン③ Pi に直接接続（完全オフライン）

```
WiFi SSID : ube-map
パスワード : ubemap2026

http://192.168.4.1/raster.html
http://192.168.4.1/vector.html
```

## 地図が表示されたら

右上の現在地ボタンを押すと GPS 位置が表示されます。位置情報の許可を求められたら「許可」を選んでください。

`http://` の URL（HTTPS ではない）でも Geolocation API は動作します。ローカルネットワーク上のアドレスはセキュアコンテキストとして扱われるためです。
